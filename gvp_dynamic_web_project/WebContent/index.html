Introduction:

The purpose of this exercise is to give you an opportunity to show how you organize and communicate in code.  Focus on good structure, clarity and simplicity rather than sophisticated implementation details.  Think of this as an initial sketch meant to help guide a team rather than a final implementation.  It should work, but you don’t need to sweat all the details.

Your assignment is to build a simple smart plate Android App to simulate a fleet of plates and a local HTTP server.  Each smart plate has a display, a GPS and is able to communicate with the server.  Each plate also has a unique id that is registered with the server.  

When a plate boots up, it sends the server its id and the server responds with a license number.  When the plate is moving, it displays this number.  Whenever the plate stops, it reports it’s location to the server and receives a message to display at that location.

Details:

  • Implement each plate as a thread in a single JVM (rather than as separate network nodes)
  • Implement the server using one or multiple threads in the same JVM.
  • The plates communicate with the server using your choice of inter-thread communication.
  • A plate displays something by printing it to the console.
  • Plate id’s are integers from 1-10
  • License numbers are 3 letters followed by a dash and 3 digits as in ABC-123
  • Instead of GPS coordinates, the plate will send symbolic locations and the server will always respond with a fixed message for a given location.  The locations are corresponding messages are as follows:
  • HomeLot -> “Ready to roll!”
  • DowntownParking -> “Check me out”
  • UptownParking -> “Rent me”
  • OutOfBounds -> “Help!  I’m lost”
  • Each plate should begin at the HomeLot, randomly travels to three other locations (repeats are fine) and then returns to the HomeLot.  It should take 10 seconds to travel from one location to another.
  • Here’s some example output for a single car/plate:
  • Plate 1 at HomeLot: Ready to roll!
  • Plate 1 is moving: AGC-388
  • Plate 1 at UptownParking: Rent me
  • Plate 1 is moving: AGC-388
  • Plate 1 at OutOfBounds: Help! I’m lost
  • Plate 1 is moving: AGC-388
  • Plate 1 at DowntownParking: Check me out
  • Plate 1 is moving: AGC-388
  • Plate 1 at HomeLot: Ready to roll!

  • Every minute, the  server should print a line to the console indicating total number of plates served and the total number of messages served
  • The final program should start the server, simulate 10 plates and remain running.  
  • Please use Java as the implementation language.
